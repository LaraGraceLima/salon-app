================================================================================
                            PHASE 1: LOGIN
================================================================================

User Device (Expo Go)
    ↓
    [LoginScreen.js]
    ├─ User enters email & password
    ├─ API call to /api/users/login
    └─ Receives: { token, name, email }
    ↓
    [Call setIsLoggedIn(token, name, email)]
    ↓
    [App.js - handleLogin()]
    ├─ setUserToken(token)
    ├─ setUserName(name)
    ├─ setUserEmail(email)
    └─ setIsLoggedIn(true)
    ↓
    [Navigation switches to AppStack]

================================================================================

App.js (State Management)
├─ userToken: "eyJhbGciOiJIUzI1NiIs..."

    ├─ initialParams: { userToken, userName, userEmail, onLogout }

                      PHASE 4: CREATE BOOKING
================================================================================

[BookingScreen.js]

================================================================================
                    API CALLS WITH TOKEN
================================================================================

1. LOGIN
   POST /api/users/login
   Body: { email, password }
   Response: { token, name, email }

2. SIGNUP
   POST /api/users/signup
   Body: { name, email, phone, password }
   Response: { success: true }
   Then: Auto-login with same credentials

3. GET STYLISTS
   GET /api/stylists
   No auth needed
   Response: [ { id, name, specialization, phone, status } ]

4. GET SERVICES
   GET /api/services
   No auth needed
   Response: [ { id, name, price } ]

5. CREATE BOOKING
   POST /api/bookings
   Headers: { Authorization: "Bearer {token}" }
   Body: { stylist_id, service_id, date_time, notes }
   Response: { success: true, bookingId }

6. GET MY BOOKINGS
   GET /api/users/bookings
   Headers: { Authorization: "Bearer {token}" }
   Response: [ { id, serviceName, stylistName, dateTime, price, status } ]

================================================================================
                    ERROR HANDLING
================================================================================

Scenario: Token is empty
├─ Check: route.params.userToken
├─ If undefined: Show error "Please login first"
├─ If empty string: Show error "Session expired"
└─ Solution: Logout and login again

Scenario: API returns 401 Unauthorized
├─ Cause: Token is invalid or expired
├─ Response: { error: "Unauthorized" }
└─ Solution: Logout and login again

Scenario: Network error
├─ Cause: No internet connection
├─ Response: Network error
└─ Solution: Retry when connection restored

Scenario: AsyncStorage error (should not happen)
├─ Cause: AsyncStorage not available
├─ Solution: Not used anymore - using React state
└─ Result: No more AsyncStorage errors

================================================================================
                    SECURITY ARCHITECTURE
================================================================================

Token Storage:
├─ Location: App.js React state (memory)
├─ Lifetime: Until app closes or logout
├─ Encryption: None needed (memory only)
└─ Clearing: Automatic on logout

Token Transmission:
├─ Method: HTTP Bearer token
├─ Header: Authorization: Bearer {token}
├─ Protocol: HTTPS (in production)
└─ Validation: Backend validates on each request

Token Validation:
├─ Backend: Verifies JWT signature
├─ Backend: Checks token expiration
├─ Backend: Validates user permissions
└─ Backend: Returns 401 if invalid
